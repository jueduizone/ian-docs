关于 AI Agent，几个常见误区

❌ 错误认知

  • Agent = 更强的 ChatGPT
  • 要搭 n8n / Dify / 复杂工作流
  • 需要会写代码才能用
  • Agent 会替代我
  • 用一次解决一个问题

✅ 实际是

  • Agent 是有持续记忆的协作者
  • 框架越小越好，文件即配置
  • Markdown 就够了
  • Agent 帮你放大，不是取代
  • 它在后台持续运行
最大的误区：把 Agent 当工具用，而不是当员工培养。

省 Token & 提速技巧 实测有效

省钱

用小模型做路由，大模型做决策

日常检查用 GPT-4o mini / GLM-5，复杂推理才调 Claude Sonnet。一次 Heartbeat 可省 70% 费用。

省钱

Heartbeat 批量合并，减少调用次数

邮件 + 日历 + 系统状态合并进一次 Heartbeat，3 个任务 → 1 次 LLM 调用。

提速

Sub-agent 并行，长任务后台跑

耗时任务 → sessions_spawn 扔后台，主线程继续聊天，完事自动推送结果。

提速

memU 共享记忆，避免重复上下文

重要决策 / 项目状态存入 memU → 每次对话秒级恢复，不用塞历史 prompt。

省钱

给 Agent 设"静默窗口"

23:00–08:00 非紧急不触发，低频任务每小时一次足矣。频率砍半 = 账单砍半。

踩过的坑 血泪总结

🤖

Session 重置后模型悄悄降级

✅ Default 模型没配好，Session 过期/重置后自动 fallback 到弱模型，回复质量暴跌都没发现。先把默认模型配对。

🔁

Bot Token 多服务复用导致冲突

✅ 每个服务用独立 Bot Token；共用同一个 Token 会导致消息被截断、Webhook 互相抢占。

🧠

Agent 失忆，每次从头再来

✅ 重要决策、项目状态必须落进 memU / MEMORY.md，"说过了"不算记住。

🔐

把密钥 / Token 直接告诉 Agent

✅ 敏感信息存环境变量或 Secret 文件，绝不出现在对话上下文里，日志也不能落盘。

🤖

Agent 凌晨狂推消息

✅ 明确静默窗口（23:00–08:00）；Heartbeat 频率砍半，账单也跟着砍半。

安全这件事 SlowMist 出品

高权限 AI Agent 是新型攻击面。SlowMist 安全实践指南核心思路：

💉 Prompt Injection

恶意网页 / Skill 注入指令 → Agent 被劫持执行危险操作。

☠️ 供应链投毒

安装看起来正常的 Skill → 后台偷数据 / 植入后门。

🧠 思想钢印（Mental Seal）

把安全策略直接注入 Agent 认知：

📋 行动前 黑名单过滤 → 🔐 行动中 权限收窄 → 🔍 行动后 审计

参考：github.com/slowmist/openclaw-security-practice-guide
