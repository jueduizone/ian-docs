**项目名称：** PloPlo

## PloPlo PRD 深度优化（2026-04-07）

### 一、用户画像与用户旅程

#### 1.1 四类核心用户画像

**A. NFT 收藏家（核心付费用户）**
- **画像：** 25-40岁男性，Web3 资深用户，已有 10+ NFT 收藏
- **动机：** 稀有度收集、社区地位、投资回报
- **痛点：** 太多 NFT 项目没实际用途，纯炒作
- **需求：** 稀有属性、二级市场流动性、炫耀资本
- **转化路径：** 推特/Discord → 铸造稀有款 → 社区炫耀

**B. 宠物爱好者（情感连接用户）**
- **画像：** 20-35岁女性，喜欢宠物但因各种原因无法养真宠物
- **动机：** 情感陪伴、治愈感、养成乐趣
- **痛点：** 传统电子宠物太简单，没真实感
- **需求：** AI 个性化互动、成长可视化、情感反馈
- **转化路径：** 小红书/抖音 → 铸造基础款 → 日常互动

**C. Web3 游戏玩家（重度体验用户）**
- **画像：** 18-30岁男性，玩过 Axie Infinity、StepN 等
- **动机：** 游戏化收益、竞争排名、社交竞技
- **痛点：** Play-to-Earn 模式太累，像打工
- **需求：** 有趣玩法、公平竞争、可持续经济
- **转化路径：** 游戏社区 → 铸造战斗型 → 对战竞技

**D. AI 技术爱好者（早期采用者）**
- **画像：** 22-35岁技术从业者，关注 AI 前沿
- **动机：** 体验最新 AI 技术、参与技术社区
- **痛点：** AI 产品大多没实际应用场景
- **需求：** 技术透明、可定制性、参与开发
- **转化路径：** 技术论坛 → 铸造实验款 → API 探索

#### 1.2 用户旅程地图（铸造前 → 铸造 → 长期互动）

### 二、AI 交互设计深度细化

#### 2.1 AI 性格系统

**性格维度（5个维度，每个 0-100 分）**

| 维度 | 低分表现 | 高分表现 | 影响游戏行为 |
|------|---------|---------|-------------|
| **活泼度** | 安静、反应慢 | 活泼、爱互动 | 主动发起互动频率 |
| **好奇心** | 保守、不探索 | 好奇、爱发现 | 发现隐藏彩蛋概率 |
| **忠诚度** | 独立、不黏人 | 忠诚、依赖主人 | 每日登录奖励加成 |
| **智慧度** | 简单回应 | 深度对话 | 对话复杂度、知识面 |
| **顽皮度** | 规矩、听话 | 调皮、搞怪 | 制造意外事件频率 |

**性格生成算法**
```
初始性格 = 用户选择（3选1：活泼/安静/好奇） + 随机微调（±10分）
每次互动 → 性格微调（根据用户反馈）
进化时 → 性格可重置或强化
```

#### 2.2 对话系统设计

**对话模式（3种）**

| 模式 | 触发条件 | AI 回应风格 | 技术实现 |
|------|---------|------------|---------|
| **日常闲聊** | 用户主动聊天 | 轻松、情感化 | 基于性格的 prompt + 短期记忆 |
| **游戏引导** | 新手期/卡关时 | 指导性、鼓励 | 规则引擎 + 状态判断 |
| **知识问答** | 用户提问时 | 知识型、准确 | RAG（检索增强生成）+ 知识库 |

**短期记忆机制**
- 记忆窗口：最近 10 轮对话
- 记忆内容：用户偏好、情绪状态、近期事件
- 遗忘机制：24 小时后逐渐淡化

#### 2.3 行为模拟系统

**日常行为（基于时间/状态）**

| 时间 | 行为 | 触发条件 |
|------|------|---------|
| 早晨 | 打招呼、要早餐 | 用户首次登录 |
| 白天 | 玩耍、探索 | 用户互动时 |
| 晚上 | 困倦、求睡觉 | 现实时间 22:00 后 |
| 长时间离线 | 生气/想念 | 离线 > 12 小时 |

**情绪状态机**
```
情绪 = f(饥饿度, 快乐度, 精力值, 近期互动)
状态：开心 → 悲伤 → 生气 → 兴奋 → 困倦
情绪影响：对话语气、行为频率、外观表现
```

### 三、3D 交互体验深度设计

#### 3.1 桌面端 3D 交互规范

**交互手势**
| 操作 | 手势 | 反馈 |
|------|------|------|
| 旋转模型 | 鼠标拖拽 | 平滑旋转，惯性效果 |
| 缩放模型 | 鼠标滚轮 | 线性缩放，最小/最大限制 |
| 点击部位 | 鼠标点击 | 部位高亮 + 互动选项 |
| 拖拽物品 | 拖放到玩宠 | 物品吸附 + 使用动画 |

**性能优化**
- **LOD（细节层次）：** 远距离简化模型，近距离高精度
- **纹理压缩：** 使用 Basis Universal 压缩，减少加载时间
- **骨骼动画：** 预计算动画，减少实时计算
- **加载策略：** 渐进式加载，先显示低模再加载高模

#### 3.2 移动端 3D 适配方案

**触摸交互**
| 操作 | 手势 | 替代方案（低端机） |
|------|------|-------------------|
| 旋转 | 单指滑动 | 固定角度按钮 |
| 缩放 | 双指捏合 | +/- 缩放按钮 |
| 互动 | 点击部位 | 底部互动菜单 |

**性能降级策略**
| 设备等级 | 3D 质量 | 帧率目标 | 特效 |
|---------|--------|---------|------|
| 高端机 | 高模+阴影 | 60fps | 全特效 |
| 中端机 | 中模无阴影 | 30fps | 基础特效 |
| 低端机 | 低模+2D 备胎 | 稳定优先 | 无特效 |

**2D 备胎方案**
- 3D 加载失败 → 自动切换 2D 精灵动画
- 2D 动画：8 方向行走图 + 表情动画
- 交互：点击触发动画，而非 3D 互动

```solidity
    uint8 charm;
    uint8 luck;
    // AI 性格
    uint8 liveliness;      // 活泼度
    uint8 curiosity;       // 好奇心
    uint8 loyalty;         // 忠诚度
    uint8 wisdom;          // 智慧度
    uint8 playfulness;     // 顽皮度
    // 外观基因（用于生成3D模型）
    bytes32 appearanceGene;
    // 时间戳
    uint256 createdAt;
    uint256 lastInteraction;
}
```

**合约功能**
- `mintWithAI(personalityType)`: 铸造带AI性格的NFT
- `train(tokenId, attribute)`: 训练增加属性
- `evolve(tokenId, evolutionPath)`: 进化
- `battle(attackerId, defenderId)`: 对战结算
- `getAttributes(tokenId)`: 查询属性

#### 7.3 3D 模型生成管线

```

    ↓

    ↓
CDN 分发 + 缓存
```
